# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes |

## Reporting A Vulnerability

Do not open a public issue for a suspected security problem.

Instead:

1. Use the repository hosting platform's private security reporting flow if it is available.
2. If private reporting is not available, contact the maintainers through a private channel before disclosure.
3. Include reproduction steps, affected files, and whether the issue impacts the skill instructions, the MCP runtime, or both.

You should receive an acknowledgment within five business days.
